Key Decisions — Locked In

Name changes in leave files = person replacement, not aliases.
"Vaishnavi Kanade (Yashwanth Sai)" means Yashwanth Sai replaced Vaishnavi. The current active name in the Team Details is the canonical key. When uploading a leave CSV, the app matches on the current active name. Old names are treated as archived/moved-away entries.
ICA Notifications are leave-aware. When the Reminders tab generates a "missed ICA usage" notification, it will skip anyone who is marked on leave that day — cross-checking leave records automatically.
Team Manager page removed from ICA tab. Team management moves entirely to the new "Team Details" section. The ICA tab's "Team Manager" sub-tab is removed; that functionality is replaced by the richer Team Details tab.

New Database Tables (Backend)

team_members

  • id, name, email, mobile
  • stream (DTW/MTC/RTR/STP/T&S)
  • role (Functional/Development/…)
  • location (Bangalore/Pune/…)
  • backup_id → FK to another team_member
  • status (active / on_leave / moved_away)
  • created_at, updated_at

leave_records

  • id, member_id → FK team_members
  • leave_date (DATE)
  • leave_type (vacation / sick / holiday)
  • month_year (e.g. "2026-06")
  • source (uploaded / manual)

holidays

  • id, holiday_date (DATE)
  • name (e.g. "Republic Day")
  • locations (JSON array of cities)
  • is_national (boolean)

Existing tables (untouched)

  • usage_records — ICA daily usage
  • users — ICA user list
  • settings — app config

The Key Integration — Leave ↔ ICA Reminders

When "Send Reminders" runs, the current flow is: find users with no ICA usage on working days → send Teams notification. The new flow adds one check:

StepOld logicNew logic
1Get all active usersGet all active team members
2Find who missed ICA usageFind who missed ICA usage
3(no check)Cross-check leave_records — remove anyone on leave that day
4(no check)Cross-check holidays — remove anyone whose location has a holiday that day
5Send notification to remaining listSend notification to remaining list (cleaner, no false alerts)

Phase 3 — What To Build Next

The Leave Monitoring section in the left sidebar currently shows a "Coming Soon" placeholder. Replace it with 3 real sub-tabs:

Sub-tab 1 — "Who's Out Today"

  • Default view — shows TODAY's date (can change date via a date-picker)
  • Two columns: On Leave (from leave_records) and On Holiday (from holidays table filtered by each person's location)
  • Each person card: name, stream, location, leave type badge (vacation/sick/holiday), their backup name + contact
  • Coverage gap alert: red warning banner if a person is out AND their backup is also out
  • Hits: GET /api/leave/today?date=YYYY-MM-DD and GET /api/coverage?date=YYYY-MM-DD

Sub-tab 2 — "Monthly Calendar"

  • Month selector (dropdown for year-month, e.g. 2026-06)
  • Grid: rows = team members, columns = days 1–30/31
  • Cell colour codes: Vacation = orange, Sick = red, Holiday (auto from their location) = blue, Weekend = grey bg, Working = white/green dot
  • Header row shows day-of-week (Mon/Tue/Wed…) and highlights weekends
  • Filter by stream
  • Hits: GET /api/leave?monthYear=2026-06 and GET /api/holidays and GET /api/roster
  • Show summary at top: total leave days this month, who had most leave, sick vs vacation split

Sub-tab 3 — "Upload Leave"

  • Upload widget for monthly leave CSV files (same style as the existing ICA CSV upload in the sidebar)
  • Month is auto-detected from filename (e.g. "(Jun-26)" → June 2026)
  • Can also upload the holiday list CSV to refresh holidays
  • Shows list of months already uploaded (query distinct month_year from leave_records)
  • Hits: POST /api/upload-leave (multipart/form-data, field name = "file") and POST /api/upload-holidays
  • Re-uploading the same month replaces that month's data (handled backend-side)

Key Notes & Gotchas

Name casing is inconsistent across files. Leave CSVs use ALL CAPS (e.g. "LOKESH VEDIUM"), team_members uses title case ("Lokesh Vedium"). The DB helper uses UPPER(member_name) for comparisons. In the frontend, use case-insensitive matching when joining leave data to roster data.
Weekends in the calendar: Row 1 of the leave CSV has day-of-week headers. For the calendar view, derive day-of-week from the actual date using JavaScript's new Date(year, month-1, day).getDay() — 0=Sun, 6=Sat.
The /api/coverage endpoint already does all the hard work for the Who's Out Today view — it returns every team member's presence/absence status, backup status, and flags coverage gaps. Just consume it.
Upload endpoint: POST /api/upload-leave accepts multipart/form-data with field name "file". Month is auto-detected from the filename (e.g. "(Jun-26)"). Returns { success, monthYear, inserted, total, members }.
